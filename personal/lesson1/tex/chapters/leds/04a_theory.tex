\subsection{Теория и интерфейсы API}

Под \hyperlink{term:api}{интерфейсом API} мы будем понимать набор функций и объектов, через которые скрипт общается с аппаратной частью дрона. Для светодиодной линейки нас интересуют прежде всего модули \hyperlink{term:ledbar}{Ledbar} и таймеры.

Модуль \texttt{Ledbar} связывает абстрактные \hyperlink{term:rgb}{RGB-компоненты} с реальными светодиодами по их \hyperlink{term:index}{индексам}. Скрипту не нужно думать о проводах и регистрах: достаточно выбрать номер светодиода и задать три числа яркости каналов.

Таймеры выступают в роли «двигателя времени» для анимаций. Вместо того чтобы блокировать выполнение с помощью \texttt{sleep()}, мы просим систему периодически вызывать нашу функцию (тик) или один раз через заданную задержку. В сочетании с \hyperlink{term:state}{состояниями} это позволяет строить сложные, но управляемые сценарии индикации.

В следующих подразделах мы:
\begin{itemize}
    \item разберём, как устроена аппаратная часть и модуль \texttt{Ledbar};
    \item рассмотрим адресацию светодиодов и связь индексов с реальными лампочками;
    \item познакомимся с основными функциями \texttt{Ledbar} для установки цвета;
    \item изучим работу таймеров и их применение для динамических эффектов.
\end{itemize}
