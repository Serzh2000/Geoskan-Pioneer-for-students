\section{Урок 3: телеметрия и диагностическая индикация}

\subsection{Цели урока}
\begin{itemize}
\item Считать базовую телеметрию в цикле и интерпретировать значения.
\item Настроить «светофор» батареи и простую реакцию на высоту по дальномеру.
\end{itemize}

\subsection{Необходимые знания}
\begin{itemize}
\item Цикл \texttt{while True}, задержка \texttt{time.sleep}.
\item Проверка \texttt{None} и пустых значений.
\end{itemize}

\subsection{Ключевые команды}
\begin{itemize}
\item \texttt{get\_battery\_status()}, \texttt{get\_dist\_sensor\_data()}, \texttt{get\_local\_position\_lps()}
\item \texttt{led\_control(led\_id=255, r, g, b)}
\end{itemize}

\begin{infobox}[Инженерная интерпретация]
\begin{itemize}
\item Телеметрия в учебных задачах нужна не для «красивого вывода», а для принятия решений.
\item LED-индикация — простой способ дебага без подключения терминала.
\end{itemize}
\end{infobox}

\begin{taskbox}[Минимальный набор упражнений]
\begin{enumerate}
\item Измените пороги напряжения в \texttt{GET\_battery\_status.py} и опишите ожидаемое поведение.
\item Добавьте в \texttt{GET\_dist\_sensor\_data.py} вывод предупреждения при расстоянии меньше 0.3 м.
\item В \texttt{GET\_local\_position.py} добавьте вычисление модуля вектора позиции.
\end{enumerate}
\end{taskbox}

\subsection{Критерии успешности}
\begin{enumerate}
\item Скрипт работает не менее 1 минуты без зависаний.
\item При потере значения датчика в консоль не выводятся ошибки трассировки.
\item Индикация LED соответствует выбранным порогам.
\end{enumerate}
