# Debug Session: blank-panel-open

Status: OPEN

## Symptom
- При открытии вкладок `Настройки` и/или `Монитор каналов` пользователь видит пустое белое окно вместо содержимого панели.

## Scope
- UI боковой панели
- Переключение вкладок через `openPanel()`
- CSS/layout для `sidebar-panels`, `sidebar-panel`, `panel-content`

## Hypotheses
1. Активная панель получает класс `active`, но реальный DOM с контентом не присутствует в документе к моменту открытия.
2. Панель активируется, но её внутренний контент имеет нулевую высоту или скрывается из-за CSS (`overflow`, `height`, `min-height`, `position`, `z-index`).
3. Открывается не та панель, либо `panelId` корректен, но соответствующая кнопка/панель конфликтует с предыдущим состоянием сайдбара.
4. Контент панели рендерится, но перекрывается другим слоем внутри контейнера `.sidebar-panels` или `.workspace-sidebar`.
5. Белое окно связано с ошибкой в последовательности `width/class toggle -> active panel -> resize refresh`, из-за чего layout пересчитывается в некорректное состояние.

## Evidence Plan
- Заинструментировать `openPanel()` и снять:
  - `panelId`
  - наличие DOM-узла панели
  - список классов панели
  - размеры `panel`, `panel-content`, `settings-stack`
  - итоговые размеры `.sidebar-panels`
  - число дочерних элементов внутри панели

## Next Step
- Добавить только runtime-инструментирование в `sidebar.ts` без изменения бизнес-логики.
